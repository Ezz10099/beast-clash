# Pixel Mage Release Build

## Purpose

The repository still contains the retired Beast Clash prototype for reference. It is not part of Pixel Mage and must never enter the Android package.

## Exact Runtime

`npm run build` creates a clean `dist/` directory containing only:

- `index.html`
- `style.css`
- `localization.css`
- `phone-polish.css`
- `arena-fx.css`
- `enemy-variety.css`
- `spell-depth.css`
- `localization.js`
- `touch-controls.js`
- `game.js`
- `phone-polish.js`
- `arena-fx.js`
- `enemy-variety.js`
- `spell-depth.js`
- `build-manifest.json`

The manifest records the release version, byte count, and SHA-256 checksum of each runtime file.

- `game.js` is bundled with the Capacitor App bridge for native pause and Android Back handling.
- Release CSS and extension JavaScript are minified with UTF-8 preserved.
- `touch-controls.js` preserves the accepted 84-canvas-pixel thumb clearance.
- `phone-polish.js`/`.css` provide the portrait shell, dashboard, responsive overlays, low-health emphasis, and protected restart.
- `arena-fx.js`/`.css` provide pointer-transparent atmosphere and combat feedback.
- `enemy-variety.js`/`.css` add current-roster enemy roles, formations, cooperation, and telegraphs.
- `spell-depth.js`/`.css` add deterministic spell synergies, named combination identities, rewrite/Spellbook previews, payoff meters, and pointer-transparent spell feedback. The layer modifies only transient combat behavior and does not access saves or the network.
- English remains the default. Arabic activation and fresh-save parameters are generated by internal tooling for tests.
- Arabic wording follows `docs/ARABIC_GLOSSARY.md`.

The build contains no remote URLs, animal assets, or legacy Phaser code, so it loads entirely offline inside Capacitor.

## Runtime Size Policy

Runtime size is measured and reported without an arbitrary total or per-file hard cap.

`npm run size:report` writes a per-file diagnostic report. Size data is used to detect accidental legacy inclusion, duplication, avoidable dependencies, or measured performance/download risks. It must not block useful development merely because code crossed an inherited prototype number.

## Repository-Only Test Tools

These internal files remain outside `RELEASE_FILES`, `dist/`, the APK, and future AAB:

- `test-launcher.html`

- `test-launcher.js`
- `cell-runner.html`

Open `test-launcher.html` through SPCK Preview for visible Arabic, English, and Cell Runner buttons. No Console input, manual URL editing, or token construction is required.

## Verification

`npm run check` runs:

- persistent workflow and Arabic-glossary integrity;
- localization, controls, portrait-polish, arena-FX, enemy-variety, and spell-depth checks;
- Cell Runner and Test Launcher checks;
- core gameplay, choice, movement, persistence, and stress checks;
- deterministic UTF-8 release construction and non-blocking runtime-size reporting;
- Android artwork and native configuration checks.

`npm run spell:check` protects combination identities, Split targeting, Freeze/Shatter timing, Echo resonance, precision/ward meters, per-Trial transient reset, Arabic presentation, phone layout, offline behavior, and release inclusion.

Before a gameplay or commercial recommendation, run `npm run evidence`. The headless evidence runtime loads both `enemy-variety.js` and `spell-depth.js`, so build, policy, movement, replay, pacing, and starting-spell results include the current gameplay layers.

Generated evidence and `dist/` are excluded from Git. Capacitor consumes a freshly verified build rather than a manually maintained copy.
