# Pixel Mage Release Scope

## Purpose

Pixel Mage is the only active game project until it reaches Google Play testing. Its job is to complete the full development-to-publishing pipeline with a small, replayable game.

## Locked Player Loop

1. Start a new run.
2. Clear five increasingly difficult waves.
3. Choose one spell upgrade after each of the first four waves.
4. Defeat the final boss or lose the run.
5. Record the score and immediately offer replay.

## Allowed Before Release

- Balance changes to the five-wave run.
- Upgrade improvements using the existing mage, bolt, slime, and boss.
- Sound, haptics, pause, settings, accessibility, and phone-control improvements.
- Android packaging, compliance, testing, store assets, and one optional rewarded revive.

## Not Allowed Before Release

- New playable characters.
- New worlds, stories, maps, shops, currencies, equipment, quests, or gacha.
- Engine changes.
- Redesigning the game into another genre.
- Starting or resuming another game project.

## Completion Gate

The project leaves active development only after it has an Android release build, passes device testing, and enters Google Play testing.
